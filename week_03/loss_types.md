# Виды лоссов для двухбашенных моделей (Лекция 3)

## Контекст: зачем нужны двухбашенные модели

Рекомендации — **многостадийный процесс**:

```
Каталог (миллионы) → [Генерация кандидатов] → Шортлист (сотни-тысячи) → [Ранжирование] → Top N (десятки)
```

**Двухбашенная модель** — модель для стадии **генерации кандидатов**. User и Item кодируются **раздельно** в эмбеддинги,
релевантность = скалярное произведение:

$$r(u, d) = \langle g_\theta(u), h_\theta(d) \rangle$$

Это факторизация вычислений: можно заранее закодировать все айтемы и использовать **ANN-индекс** (HNSW) для быстрого поиска top-K.

## 1. Softmax Loss (Full Softmax)

**Слайды 15–19**

### Интуиция

Меняем постановку задачи:
- **Раньше (ранжирование):** пользователю `u` показали видео `i`. Поставил ли он лайк ($y = 1$)?
  → $P(y=1 \mid \text{user}, \text{item})$
- **Теперь (классификация):** знаем, что пользователь `u` поставил лайк. **Какому** видео?
  → $P(\text{item} \mid \text{user}, y=1)$

Это задача **экстремальной классификации** — каждый айтем из каталога = отдельный класс.

### Формула

$$\mathcal{L}_{\text{softmax}} = -\sum_{u \in U} \log \frac{\exp(s_\theta(u, d^+))}{\sum_{j \in \mathcal{D}} \exp(s_\theta(u, j))}$$

### Свойства (слайд 28)
- Софтмакс **неявно майнит хард негативы** — вклад в градиент больше у айтемов, на которых модель ошибается
- Негативы **конкурируют между собой** за вклад в градиент
- **Глобальное сравнение** — нет проблемы folding (слайд 13)

### Проблема
Полный софтмакс **невычислим** при каталоге > 10⁵ айтемов — слишком дорогой знаменатель (и по вычислениям, и по памяти).

## 2. BCE Loss (Binary Cross-Entropy) — Pointwise

**Из ноутбука ДЗ2 (задание 5)**

### Интуиция

Самый простой подход — бинарная классификация: для каждой пары (юзер, айтем) предсказываем "релевантно или нет".
- Позитив `i⁺` → σ(r(u, i⁺)) → 1
- Негативы `i⁻` (случайные из каталога) → σ(r(u, i⁻)) → 0

### Формула

$$\mathcal{L}_{\text{BCE}} = -\left[\log \sigma(r(u, i^+)) + \sum_{i^- \in \mathcal{I}^-} \log(1 - \sigma(r(u, i^-)))\right]$$

Где $\sigma(x) = \frac{1}{1+e^{-x}}$ — сигмоида, $\mathcal{I}^-$ — сэмплированные негативы.

### Код (PyTorch)

```python
r_pos = (user_repr * E[inputs["label"]]).sum(dim=-1)
neg_ids = torch.randint(0, num_items, (batch_size, num_neg), device=device)
r_neg = (user_repr.unsqueeze(1) * E[neg_ids]).sum(dim=-1)
loss = -F.logsigmoid(r_pos).mean() - F.logsigmoid(-r_neg).mean()
```

### Свойства
- **Дешёвый** — не нужен весь каталог, только сэмплированные негативы
- **Слабое качество** — каждый пример рассматривается отдельно, нет сравнения между айтемами

## 3. BPR Loss (Bayesian Personalized Ranking) — Pairwise

**Из ноутбука ДЗ2 (задание 6)**

### Интуиция

Вместо бинарной классификации — **попарное сравнение**: скор позитива должен быть выше скора негатива.

### Формула

$$\mathcal{L}_{\text{BPR}} = -\sum_{u \in \mathbf{U}} \log \sigma(r(u, i^+) - r(u, i^-))$$

- Один позитив + один случайный негатив на юзера
- Учим модель, чтобы **разница** скоров была положительной

### Код (PyTorch)

```python
r_pos = (user_repr * E[inputs["label"]]).sum(dim=-1)
neg_ids = torch.randint(0, num_items, (batch_size,), device=device)
r_neg = (user_repr * E[neg_ids]).sum(dim=-1)
loss = -F.logsigmoid(r_pos - r_neg).mean()
```

### Свойства
- **Дешёвый** — один негатив на юзера
- **Лучше BCE** — явно сравнивает позитив с негативом
- **Хуже softmax** — видит только одну пару, а не весь каталог

## 4. Sampled Softmax Loss

**Слайд 20**

### Интуиция
Аппроксимируем знаменатель — вместо суммирования по **всему** каталогу, сэмплируем подмножество негативов.

### Формула

$$\mathcal{L}_{\text{sampled}} = -\log \frac{\exp(s_\theta(u, d^+))}{\exp(s_\theta(u, d^+)) + \sum_{d_i \in N} \exp(s_\theta(u, d_i))}$$

где $N = \{d_1, \ldots, d_{|N|} : d_i \sim Q(d)\}$ — сэмплированные негативы из распределения $Q$.

Это тоже задача классификации, но с **подмножеством** каталога в качестве классов.

## 3. InfoNCE (Contrastive Loss)

**Слайд 21**

### Интуиция
**Contrastive learning** — сближаем похожие объекты и отдаляем непохожие. В батче из $n$ положительных пар $(u_1, d_1), \ldots, (u_n, d_n)$:

### Формула

$$\mathcal{L}_{\text{InfoNCE}} = -\sum_{i=1}^{n} \log \frac{\exp(s_\theta(u_i, d_i) / \tau)}{\sum_{j=1}^{n} \exp(s_\theta(u_i, d_j) / \tau)}$$

где $\tau > 0$ — **температура**, использует **in-batch negatives**.

---

## Источники негативов

**Слайды 31–33**

### Uniform negatives (слайд 32)
- $d \sim \text{Uniform}(\mathcal{D})$ — равномерное сэмплирование из каталога
- Очень **"простые"** для модели → малый вклад в градиент
- Нужно **большое количество** негативов

### In-batch negatives (слайды 22, 33)
- Для каждой пары **переиспользуем** позитивы других пар как негативы
- Эквивалентно сэмплированию из **униграммного** (частотного) распределения
- Более **сложные** негативы (популярные айтемы чаще попадают)
- Нужно **меньше** негативов для обучения
- **Эффективно**: не нужно перевычислять эмбеддинги для негативов

### Mixed Negative Sampling (слайд 31)
- Комбинация uniform + in-batch негативов
- Ссылка: *Mixed Negative Sampling for Learning Two-tower Neural Networks in Recommendations*

## LogQ Correction

**Слайды 34–41**

### Проблема in-batch негативов (слайд 35)
In-batch негативы приводят к **систематическому смещению**:
- Популярные айтемы **часто** появляются в негативах
- Один раз как позитив и `batch_size - 1` раз как негатив
- Модель видит корреляцию популярность ↔ "негативность" → **штрафует популярные объекты**

### Решение: LogQ коррекция (слайд 39)

$$\mathcal{L}_{\text{logQ}} = -\log \frac{\exp(s_\theta(u, d^+) - \log Q(d^+))}{\sum_{d_i \in N} \exp(s_\theta(u, d_i) - \log Q(d_i))}$$

- $Q$ — распределение, из которого сэмплируются негативы
- **Интуиция**: logQ поправка **штрафует айтемы за популярность** при обучении, убирая смещение
- Ссылки: *Sampling-Bias-Corrected Neural Modeling for Large Corpus Item Recommendations*

## Дополнительно: косинус и температура

### Косинус (слайд 23)
- Обучение с косинусом **более стабильное** (логиты ограничены по масштабу)
- Нормы айтемов выучивают **популярность** → косинус убирает влияние нормы (понижаем popularity bias)

### Температура (слайды 24–30)
- При косинусе логиты ∈ [-1, 1] → софтмакс слишком "гладкий"
- Температура $\tau$ **масштабирует** логиты: $s/\tau$
- Чем **меньше** температура → больше **uniformity** (равномерное покрытие сферы)
- Чем **больше** температура → больше **alignment** (похожие объекты ближе)
- Температура = **мера неопределённости** (Boltzmann / Gibbs distribution)
- Можно: фиксировать / делать расписание / делать **обучаемой**

## Сравнительная таблица лоссов (из лекции + ДЗ)

| Loss            | Тип          | Негативы                     | Масштабируемость | Качество      |
| --------------- | ------------ | ---------------------------- | ---------------- | ------------- |
| Full Softmax    | Softmax      | Весь каталог                 | ❌ Дорого         | ✅ Лучшее      |
| BCE             | Pointwise    | Uniform (сэмплированные)     | ✅ Дёшево         | ⚠️ Низкое     |
| BPR             | Pairwise     | Uniform (сэмплированные)     | ✅ Дёшево         | ⚠️ Среднее    |
| Sampled Softmax | Softmax      | Uniform (подмножество)       | ✅ Среднее        | ✅ Хорошее     |
| InfoNCE         | Contrastive  | In-batch                     | ✅ Эффективно     | ✅ Хорошее     |
| InfoNCE + logQ  | Contrastive  | In-batch + коррекция         | ✅ Эффективно     | ✅ Отличное    |

> **Ключевой вывод из лекции**: Full softmax — идеальный лосс, потому что область обучения (весь каталог)
> совпадает с областью применения (top-K из всего каталога). Все остальные лоссы — аппроксимации
> для случая, когда полный софтмакс слишком дорог.
